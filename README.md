## 三、未来规划

### 短期（立即可做）
1. **技能字段高度适配** — 表单 skills 字段仅 44pt 高，4 行 bullet 可能溢出，需动态扩展
2. **添加 "applicant_name" → "英文名" 映射** — 消除当前唯一的映射盲区
3. **mapping 中增加 `获奖` 字段** — 当前表单不含 awards 字段，若 PDF 包含需映射

### 中期（功能增强）
4. **集成 LLM 语义映射引擎** — 在映射引擎第 4 级之后增加 LLM fallback，处理规则无法覆盖的语义等价（如 "现住址"→"address"、"毕业院校"→"university"）
5. **签名/照片坐标嵌入** — 检测 "签名" / "照片" 关键字，从 image_files 中插入对应图片
6. **MinerU / Zerox 作为扫描件备选路径** — 当 PDF 为纯扫描件（无文本层）时自动切换
7. **LLM 驱动的资料提取** — 替换正则引擎，用 LLM 从自由格式简历中结构化提取

### 长期（架构升级）
8. **嵌套表格与多栏布局支持** — 集成 deepdoctection 完整文档分析框架
9. **本地化 LLM 部署** — 使用量化模型（如 Ollama / llama.cpp）实现纯本地运行，解决隐私顾虑
10. **GUI 前端** — 开发桌面端图形界面，拖拽式操作
11. **批量处理** — 支持一次处理多份 PDF 表单
12. **大规模测试集构建** — 覆盖扫描件、手写体混合、多语言等真实场景
